# Normalize Monorepo

Normalize is a polyglot code intelligence CLI toolchain providing structural analysis of codebases through AST-based extraction. The monorepo contains 30+ Rust crates (published on crates.io), a VS Code extension, a web sessions viewer, documentation site, and supporting tooling. The main `normalize` binary consumes a library ecosystem of domain crates covering language support, facts extraction, rules evaluation, manifest parsing, and output formatting. Core development conventions are in `CLAUDE.md`; architecture decisions, design philosophy, and CLI documentation live under `docs/`.

The `normalize-syntax-rules` crate provides the syntax linting engine; 39 builtin rules cover Rust (11), JS/TS (8), Python (7), Go (4), Ruby (4), and cross-language (3), with severity promotion — error for genuine bugs (mutable default args, bare except, dbg!/todo! macros), warning for style — and fix-fixture infrastructure for testing auto-fix transforms. `run_rules()` takes separate `project_root` and `root` (scan target) parameters so allow-list path matching works correctly when scanning a subdirectory.

Every directory with source files has a `SUMMARY.md` — enforced at `severity=error` via `normalize rules run --engine native` in the pre-commit hook (native engine covers stale-summary, check-refs, stale-docs, check-examples). The `service/` layer (server-less `#[cli]`) is the primary CLI registration point; all `cmd_*` i32-returning wrappers have been eliminated from both `commands/` and `normalize-rules/src/` — service methods return `Result<T, String>` with natural `?` propagation. `normalize analyze check` deleted — subsumed by `normalize rules run --engine native`. Rule orchestration (`RulesService`, `run_rules_report()`, `apply_native_rules_config()`, syntax rule runner) has been extracted into the `normalize-rules` crate. `normalize rules list` returns a structured `RulesListReport` (Serialize+JsonSchema) covering all three engines (syntax/fact/native), with `OutputFormatter` pretty/compact rendering. `main()` is `#[tokio::main] async fn`; service methods are `async fn`; sync-trait bridge code (`path_resolve.rs`, `extract.rs`, `health.rs`) uses `block_in_place`+`Handle::try_current` instead of `Runtime::new()`.

Three sub-crates (`normalize-facts`, `normalize-filter`, `normalize-syntax-rules`) now ship standalone CLI binaries behind a `cli` feature, with `dep:clap` dropped (server-less re-exports clap). Service framework upgraded to server-less 0.4.0; `about =` renamed to `description =` in all `#[cli]` attributes. The main `normalize` crate uses `cli-full = ["cli", "jq-cli", "rg-cli", "ast-grep-cli"]` as default, with drop-in wrappers each implying `cli`.

Language query coverage in `normalize-languages/src/queries/`: ~325 `.scm` files covering 69 languages across 5 query types (tags, calls, complexity, imports, types). Import extraction added for HTML (`<script src>`, `<link href>`) and Jinja2 (`extends`/`import`/`from`/`include`) via `.imports.scm` query files. `grammars/jinja2/` — high-quality Jinja2 grammar written from scratch (54 corpus tests, full expression language, distinct named node types per statement kind); installed locally, pending publish as our own crate. All query files are registered in `grammar_loader.rs`; TSX uses the TypeScript imports query. `docstring.rs` adds `extract_preceding_prefix_comments` shared by Lua and R. `body.rs` `analyze_brace_body`/`analyze_paren_body` delegate to a shared `analyze_delimited_body`. `arch_common.rs` in `normalize-package-index` adds `parse_arch_db`/`parse_arch_db_desc` shared by CachyOS and EndeavourOS. `DepsExtractor` uses a query-first import path (`collect_imports_from_query` via `*.imports.scm`) with trait-based fallback. `type_refs` table stores deep type edges for Rust, TypeScript/TSX, Python, Go, Java, C#, Kotlin, Swift, C++, Ruby. Extraction bugs fixed for Elixir (no-args defs), Haskell (dedup), Kotlin (property_declaration short-circuit). Multi-repo support (`--repos-dir`) on all analyze and tools commands — standardized flag name across ownership, hotspots, duplicates, contributors, activity, repo_coupling, cross_repo_health, tools lint/test. Rule configuration migrated from `[analyze.rules]` to `[rules]`; `normalize-rules-config` holds canonical types. 257 fixture tests across 68 languages.

`normalize-filter` detects bare language names passed to `--only`/`--exclude` (e.g. `--only rust`) and emits a helpful error instead of silently matching nothing. `normalize analyze complexity` and `normalize analyze length` now support single-file input, routing to `analyze_file_*` directly instead of requiring a directory. `normalize analyze complexity-trend`, `length-trend`, `density-trend`, `test-ratio-trend` — scalar time-series commands using git worktrees. `normalize analyze complexity` and `length` support `--diff <ref>` for annotated delta views against a baseline commit. Import extraction fixes: Groovy `groovy.imports.scm` updated to use `import: (qualified_name)` field accessor (was using invalid `dotted_identifier` node kind); `DepsExtractor::collect_imports_from_query` now skips empty-path sentinel matches and returns `None` when empty (fixing Scala ghost imports and ensuring correct trait fallback for both languages).

`normalize config schema/show/validate/set` — generic schema-driven config UI. `show` walks the JSON Schema to display all available options with descriptions and types; `--section` accepts dotted paths; `--set-only` hides unset fields. `set` validates against the schema before writing and blocks on errors; `--force` bypasses; `--dry-run` previews. Uses `jsonschema` crate for validation, `toml_edit` for typed writes. Updated 2026-03-10.
