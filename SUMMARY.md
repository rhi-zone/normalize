# Normalize Monorepo

Normalize is a polyglot code intelligence CLI toolchain providing structural analysis of codebases through AST-based extraction. The monorepo contains 30+ Rust crates (published on crates.io), a VS Code extension, a web sessions viewer, documentation site, and supporting tooling. The main `normalize` binary consumes a library ecosystem of domain crates covering language support, facts extraction, rules evaluation, manifest parsing, and output formatting. Core development conventions are in `CLAUDE.md`; architecture decisions, design philosophy, and CLI documentation live under `docs/`.

The `normalize-syntax-rules` crate provides the syntax linting engine; 39 builtin rules cover Rust (11), JS/TS (8), Python (7), Go (4), Ruby (4), and cross-language (3), with severity promotion — error for genuine bugs (mutable default args, bare except, dbg!/todo! macros), warning for style — and fix-fixture infrastructure for testing auto-fix transforms. `run_rules()` takes separate `project_root` and `root` (scan target) parameters so allow-list path matching works correctly when scanning a subdirectory.

Every directory with source files has a `SUMMARY.md` — enforced at `severity=error` via `normalize rules run --engine native` in the pre-commit hook (native engine covers stale-summary, check-refs, stale-docs, check-examples). The `service/` layer (server-less `#[cli]`) is the primary CLI registration point; dead `cmd_*` i32-returning wrappers were removed from `commands/analyze/` and `commands/rules.rs` — service methods call analysis functions directly. Rule orchestration (`RulesService`, `run_rules_report()`, `apply_native_rules_config()`, syntax rule runner) has been extracted into the `normalize-rules` crate.

Three sub-crates (`normalize-facts`, `normalize-filter`, `normalize-syntax-rules`) now ship standalone CLI binaries behind a `cli` feature, with `dep:clap` dropped (server-less 0.3.1 re-exports clap). The main `normalize` crate uses `cli-full = ["cli", "jq-cli", "rg-cli", "ast-grep-cli"]` as default, with drop-in wrappers each implying `cli`.
