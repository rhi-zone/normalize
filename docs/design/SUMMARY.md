# Design

Design documents for features and architectural changes. Topics include the analyze command consolidation (collapsing 50+ subcommands into a composable query space), rules unification, LSP diagnostics (now implemented with two-tier architecture: immediate syntax, debounced fact rules), duplicate detection, syntax linting architecture, session refactoring, shadow-git support, test coverage gaps, tags.scm symbol extraction migration, semantic retrieval (vector embeddings over structurally-derived chunks for acceleration-structure-style code navigation), CLI coherence audit (systematic review of naming, flag, and behavioral inconsistencies across all subcommands), and daemon-as-kernel (architectural direction for the daemon as the codebase's kernel — injected trait APIs, incremental index maintenance, command taxonomy, lifecycle, and migration path). These are working documents — they evolve alongside implementation and may reflect proposals at various stages of completion.
