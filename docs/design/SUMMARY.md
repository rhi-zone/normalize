# Design

Design documents for features and architectural changes. Topics include the analyze command consolidation (collapsing 50+ subcommands into a composable query space), rules unification, LSP diagnostics (now implemented with two-tier architecture: immediate syntax, debounced fact rules), duplicate detection, syntax linting architecture, session refactoring, shadow-git support, test coverage gaps, and tags.scm symbol extraction migration. These are working documents — they evolve alongside implementation and may reflect proposals at various stages of completion.
