# Docs

Project documentation for the normalize toolchain. Top-level files cover architecture decisions (`architecture-decisions.md`), design philosophy (`philosophy.md`), CLI design and surface (`cli-design.md`), language support (`language-support.md`), dogfooding notes (`dogfooding.md`), rule specifications (`rules.md`, `syntax-rules.md`, `fact-rules.md`), the external native rule protocol (`rules-external-protocol.md`), session modes (`session-modes.md`), CLI usability findings (`usability.md`), agent query design (`agent-query.md`), agent UX audit (`agent-ux-audit.md`), CI command documentation (`ci.md`), an adversarial architecture audit (`audit-2026-03-12.md`), and a performance baseline (`perf-baseline.md`) tracking wall-clock and memory benchmarks for the 0.3.0 milestone. Subdirectories group CLI command references (`cli/`), in-progress design proposals (`design/`), research notes (`research/`), workflow pattern specs (`workflows/`), archived historical design docs (`archive/`), and MkDocs stylesheet overrides (`stylesheets/`). The directory also contains MkDocs package configuration for building the documentation site.
