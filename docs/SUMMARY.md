# Docs

Project documentation for the normalize toolchain. Top-level files cover architecture decisions (`architecture-decisions.md`), design philosophy (`philosophy.md`), CLI design and surface (`cli-design.md`), language support (`language-support.md`), dogfooding notes (`dogfooding.md`), rule specifications (`rules.md`, `syntax-rules.md`, `fact-rules.md`), the external native rule protocol (`rules-external-protocol.md`), session modes (`session-modes.md`), CLI usability findings (`usability.md`, `cli-ux-audit.md`), agent query design (`agent-query.md`), agent UX audit passes (`agent-ux-audit.md`, `agent-ux-audit-2.md`), CI command documentation (`ci.md`), an adversarial architecture audit (`audit-2026-03-12.md`), performance baseline (`perf-baseline.md`), CLI drop-in integrations (`cli-dropin-integrations.md`), context redesign (`context-redesign.md`), documentation guide (`documentation.md`), edit paradigm comparison (`edit-paradigm-comparison.md`), language capability traits (`language-capability-traits.md`), lint architecture (`lint-architecture.md`), LLM evaluation and comparison (`llm-evaluation.md`, `llm-comparison.md`, `llm-code-consistency.md`, `langgraph-evaluation.md`), locals.scm coverage (`locals-scm.md`), log analysis (`log-analysis.md`), manifest support (`manifest-support.md`), primitives spec (`primitives-spec.md`), prior art (`prior-art.md`), repository coverage (`repository-coverage.md`), ast-grep implementation retro (`retro-ast-grep-implementation.md`), tools documentation (`tools.md`), unification notes (`unification.md`), and view filtering (`view-filtering.md`). `index.md` and `introduction.md` serve as documentation site entry points. Subdirectories group CLI command references (`cli/`), in-progress design proposals (`design/`), research notes (`research/`), workflow pattern specs (`workflows/`), archived historical design docs (`archive/`), VitePress site configuration (`.vitepress/`), and MkDocs stylesheet overrides (`stylesheets/`). The directory also contains package configuration (`package.json`, `bun.lock`, `pnpm-lock.yaml`) for building the documentation site.
