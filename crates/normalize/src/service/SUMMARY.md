# src/service

Server-less `#[cli]` service layer — the primary CLI registration point. Every `#[cli]` method carries an explicit `display_with = "..."` attribute routing to `OutputFormatter`; orphaned `impl Display` blocks have been removed. The `NormalizeService` root struct composes sub-services: `AnalyzeService`, `ConfigService`, `DaemonService`, `EditService`, `FactsService` (CLI name `structure`), `GrammarService`, `GuideService`, `GenerateService`, `PackageService`, `RankService`, `RatchetService`, `RulesService` (from `normalize-rules`), `ServeService`, `SessionsService`, `SyntaxService`, `ToolsService`, `ViewService`. Helper methods live in separate impl blocks; JSON/schema output is generated automatically by the proc macro.

Report type naming convention: all report structs use the `Report` suffix. `DaemonRunReport` is the return type of `daemon run` (foreground execution); `DaemonActionReport` covers lifecycle actions (start/stop); `DaemonRootReport` covers root management (add/remove). Report structs do not carry a `success: bool` field — errors are signalled via `Err(...)` in the `Result` return type. The single-method display bridge pattern (`display_X` wrapping `format_text`/`format_pretty`) has been replaced: all services that needed pretty-routing now use a single `display_output<T: OutputFormatter>` generic method.

`PackageService` (`package.rs`) now returns typed report structs (`PackageInfoReport`, `PackageListReport`, `PackageTreeReport`, `PackageWhyReport`, `PackageOutdatedReport`, `PackageAuditReport`) backed by `normalize_ecosystems` types; the old `PackageReport { success, data }` hollow wrapper is gone. `ConfigService` (`config.rs`) provides `normalize config schema/show/validate/set`; `schema()` now returns `ConfigSchemaReport` with a `format_text()` impl. `RankService` (`rank.rs`) hosts all ranked-list commands. `ViewService` (`view.rs`) hosts `view` plus graph navigation subcommands; `trace()` returns `TraceReport`. `SessionsService` (`sessions.rs`) supports `--mode` on list/stats/messages/analyze. `RatchetService` (`ratchet.rs`) is a thin re-export of `normalize_ratchet::service::RatchetService`. The `normalize ci` method runs all three rule engines in sequence and exits non-zero on errors; flags `--no-syntax`, `--no-native`, `--no-fact`, `--strict`, `--sarif`.
