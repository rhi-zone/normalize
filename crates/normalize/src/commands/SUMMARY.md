# src/commands

CLI command implementations, one module per top-level subcommand. Most modules contain computation logic, report structs, and `OutputFormatter` implementations for their command; the service layer in `src/service/` wires these into the `#[cli]`-generated CLI. Exception: `edit` contains only shared type definitions (`Position`, `Operation`, `EditAction`, `EditChange`, `EditResult`) — all edit computation logic lives in `src/service/edit.rs`. Top-level modules: `analyze`, `context`, `daemon`, `edit`, `facts`, `find_references`, `generate`, `grammars`, `history`, `init`, `package`, `sessions`, `text_search`, `tools`, `translate`, `update`, `view`. Shared helper `build_filter()` constructs `Filter` values from `--exclude`/`--only` flags. `init` includes an interactive rule setup wizard (`normalize init --setup`) that walks the user through enabling/disabling rules based on live violation counts. `grammars` and `history` expose only report types with public constructors; `package` exposes `PackageAction` and `run_package_action()` for direct use by `service/package.rs`. **Rule orchestration has been extracted to the `normalize-rules` crate**: `run_rules_report()`, `apply_native_rules_config()`, `RulesService`, and the syntax rule runner are now in `normalize-rules`; callers use `normalize_rules::run_rules_report()` and `normalize_rules::RulesService` directly. `sessions` subcommand added: `show`, `list`, `stats`, `analyze` for session-level analytics. All `cmd_*` wrappers (vestigial i32-returning functions) have been eliminated — command modules now expose `build_*` or descriptively-named functions directly consumed by the service layer. `analyze/ceremony.rs`, `analyze/docs.rs`, and `view/search.rs` updated to use `lang.as_symbols().is_some()` (capability query) instead of the removed `lang.has_symbols()` bool method.
