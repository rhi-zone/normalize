# src/commands

CLI command implementations, one module per top-level subcommand. Most modules contain computation logic, report structs, and `OutputFormatter` implementations for their command; the service layer in `src/service/` wires these into the `#[cli]`-generated CLI. Exception: `edit` contains only shared type definitions (`Position`, `Operation`, `EditAction`, `EditChange`, `EditResult`) — all edit computation logic lives in `src/service/edit.rs`. Top-level modules: `analyze`, `context`, `daemon`, `edit`, `facts`, `find_references`, `generate`, `grammars`, `history`, `init`, `package`, `rules`, `sessions`, `text_search`, `tools`, `translate`, `update`, `view`. Shared helper `build_filter()` constructs `Filter` values from `--exclude`/`--only` flags. `init` includes an interactive rule setup wizard (`normalize init --setup`) that walks the user through enabling/disabling rules based on live violation counts. `grammars` and `history` expose only report types with public constructors (no `cmd_*_service` indirection); `package` exposes `PackageAction` and `cmd_package` for direct use by `service/package.rs`.
