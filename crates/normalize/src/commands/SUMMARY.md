# src/commands

CLI command implementations, one module per top-level subcommand. Each module defines report structs, implements `OutputFormatter` (`format_text` / `format_pretty`), and exposes computation functions consumed directly by the service layer in `src/service/`. Top-level modules: `analyze`, `ci`, `context`, `daemon`, `edit`, `facts`, `find_references`, `generate`, `grammars`, `history`, `init`, `package`, `sessions`, `syntax`, `text_search`, `tools`, `translate`, `update`, `view`.

Notable modules: `ci.rs` defines `CiReport` (wraps `DiagnosticsReport` with `engines_run` and `duration_ms`, with accessor methods `error_count`/`warning_count`/`info_count` computed on demand via a custom `Serialize` impl); `edit.rs` contains only shared type definitions (`Position`, `Operation`, `EditAction`, `EditChange`, `EditResult`) — all edit computation lives in `src/service/edit.rs`; `package.rs` exposes `PackageAction` and `run_package_action()` for direct use by `service/package.rs`; `init.rs` includes an interactive rule setup wizard (`normalize init --setup`); `sessions/` handles session-level analytics (`show`, `list`, `stats`, `analyze`, `messages`). Rule orchestration (`RulesService`, `run_rules_report()`, `apply_native_rules_config()`, syntax rule runner) lives in the `normalize-rules` crate, not here. The shared helper `build_filter()` constructs `Filter` values from `--exclude`/`--only` flags.
