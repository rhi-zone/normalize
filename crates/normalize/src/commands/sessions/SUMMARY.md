# src/commands/sessions

`normalize sessions` subcommand implementations for inspecting Claude Code and other agent session logs. Modules: `sort` (shared sort infrastructure — `SortDir`, `SortKey<F>`, `SortSpec<F>`, `DefaultDir` trait; each subcommand defines its own `SortField` enum implementing `DefaultDir + FromStr`), `list` (enumerate sessions with fuzzy matching, `SessionListReport`; `--sort` accepts `date`/`timestamp` (desc default), `duration` (desc), `name`/`session` (asc)), `show` (display a single session's turns, `SessionShowReport`), `messages` (extract message content; default output groups consecutive messages under a per-session header showing `[id] project  date`, each message indented as `  [role] HH:MM:SS  text`; `--sort` accepts a comma-separated list of sort keys optionally prefixed with `-` (desc) or `+` (asc) — supported fields: `tokens` (numeric, default desc), `timestamp` (date, default desc), `session` (string, default asc); e.g. `--sort -tokens` for heaviest turns first, `--sort timestamp` for chronological interleaved view, `--sort +session,-tokens` for multi-key sort; when `timestamp` is a sort key the display switches to flat interleaved mode; `--role` accepts user/assistant/tool/system/all; line-mode via `--grep --context` groups by turn; `--sequence Grep,Grep,Read` filters to turns where consecutive tool calls match the given pattern (case-insensitive prefix match on tool name) with `--context-turns N` (default 1) turns of surrounding context; `SortField` + `SortSpec<SortField>` in `messages.rs` (re-exports `SortDir`/`SortKey`/`SortSpec` from `sort.rs`); `MessagesReport`), `stats` (token/turn statistics; `--sort` accepts `calls` (desc default), `errors` (desc), `name` (asc) — sorts the per-tool rows in the output; wires through `tool_sort` hint on `SessionAnalysisReport`; `extract_repo_name` resolves Claude Code's dash-separated directory names back to real project names via `resolve_claude_dir_name` — a cached greedy path-walking algorithm that handles projects with dashes in their name), `patterns` (Markov chain transition matrix analysis of tool call sequences across sessions — computes population-level and per-session transition matrices, outlier detection via Frobenius norm divergence; `--sort` accepts `divergence` (desc default), `tool_count` (desc), `turn_count` (desc), `path` (asc) — sorts the outlier rows; `PatternsReport`), `plans` (extract plan-mode content, `PlansListReport`), `subagents` (list and summarize subagent sessions for a parent session, `SubagentsReport`), `analyze` (delegate to normalize-session-analysis), `serve` (optional web UI, gated on `sessions-web` feature). Session path resolution supports literal, prefix, fuzzy matching via `nucleo-matcher`, and subagent ID lookup. `mod.rs` uses `gix` (pure-Rust) to discover git root for project directory resolution (no `git` binary required). The `--mode` flag (interactive/subagent/all) on `list`, `stats`, `messages`, `analyze`, `patterns` controls which session types are included. `SessionMode` enum handles comma-delimited mode values. `TruncationInfo` (defined in `mod.rs`) carries metadata when `--limit` truncates results — reports include it in both text output (human-readable notice) and JSON output (structured `truncated` field).
