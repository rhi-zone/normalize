# src/commands/sessions

`normalize sessions` subcommand implementations for inspecting Claude Code and other agent session logs. Modules: `list` (enumerate sessions with fuzzy matching, `SessionListReport`), `show` (display a single session's turns, `SessionShowReport`), `messages` (extract message content; default sort groups consecutive messages under a per-session header showing `[id] project  date`, each message indented as `  [role] HH:MM:SS  text`; `--sort timestamp` interleaves messages from all sessions chronologically, showing `[id] [role] YYYY-MM-DD HH:MM:SS  text` with no session headers; `--role` accepts user/assistant/tool/system/all; line-mode via `--grep --context` groups by turn; `SortOrder` enum: `Session` (default), `Timestamp`; `MessagesReport`), `stats` (token/turn statistics), `patterns` (Markov chain transition matrix analysis of tool call sequences across sessions — computes population-level and per-session transition matrices, outlier detection via Frobenius norm divergence, `PatternsReport`), `plans` (extract plan-mode content, `PlansListReport`), `subagents` (list and summarize subagent sessions for a parent session, `SubagentsReport`), `analyze` (delegate to normalize-session-analysis), `serve` (optional web UI, gated on `sessions-web` feature). Session path resolution supports literal, prefix, fuzzy matching via `nucleo-matcher`, and subagent ID lookup. The `--mode` flag (interactive/subagent/all) on `list`, `stats`, `messages`, `analyze`, `patterns` controls which session types are included. `SessionMode` enum handles comma-delimited mode values. `TruncationInfo` (defined in `mod.rs`) carries metadata when `--limit` truncates results — reports include it in both text output (human-readable notice) and JSON output (structured `truncated` field).
