# src/commands/view

`normalize view` subcommand implementations. Provides a unified view of files, directories, and symbols extracted from the index. Modules: `file` (render a file with syntax context), `symbol` (show a single symbol with docstring/signature/children), `tree` (directory/module tree), `lines` (line-range view), `history` (git history for a path or symbol), `search` (fuzzy symbol search across the index, `search_symbols`), `report` (shared `ViewReport` struct). Configuration via `ViewConfig` (depth, line numbers, show_docs defaults). The `--full` flag emits raw file source; `--types-only` filters to type-bearing symbols.
