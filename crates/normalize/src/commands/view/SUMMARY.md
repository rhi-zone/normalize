# src/commands/view

`normalize view` subcommand implementations. Provides a unified view of files, directories, and symbols extracted from the index. Modules: `file` (render a file with syntax context), `symbol` (show a single symbol with docstring/signature/children), `tree` (directory/module tree), `lines` (line-range view), `history` (git history for a path or symbol — now exposed as `normalize view history` subcommand via `ViewService`; `build_view_history_report` returns `ViewHistoryReport` directly), `search` (fuzzy symbol search across the index, `search_symbols`), `report` (shared `ViewReport` struct with `warnings` field for unsupported-language feedback). Configuration via `ViewConfig` (depth, line numbers, show_docs defaults). The `--full` flag emits raw file source; `--types-only` filters to type-bearing symbols. Data/config languages (JSON, YAML, TOML, HTML, CSS, XML) surface a warning when viewed, rather than silently returning empty symbols. `ViewOutput::History` variant was removed — history is now a dedicated subcommand with `ViewHistoryReport` as its output type.
