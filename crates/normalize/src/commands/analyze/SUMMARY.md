# src/commands/analyze

All `normalize analyze` subcommand implementations (~49 files). Each file defines a report struct, implements `OutputFormatter` (`format_text` / `format_pretty`), and exposes a computation function. Representative subcommands: `complexity`, `length`, `hotspots`, `coupling`, `coupling_clusters`, `duplicates`, `docs`, `stale_docs`, `stale_summary`, `imports`, `layering`, `ownership`, `security`, `trace`, `trend`, `call_graph`, `graph`, `fragments`, `skeleton_diff`, `check_refs`, `check_examples`, `rules_cmd`, `summary`. `mod.rs` declares all modules and defines the shared `AnalyzeConfig` struct. `walk.rs` provides `gitignore_walk()` — a shared gitignore-aware file walker used by `check_refs`, `stale_docs`, `check_examples`, and `stale_summary`; respects `.gitignore` via the `ignore` crate and prunes well-known dependency/build directories (`node_modules`, `vendor`, `target`, etc.).
