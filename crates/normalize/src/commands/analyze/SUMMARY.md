# src/commands/analyze

All `normalize analyze` and `normalize rank` subcommand implementations (~48 files). Each file defines a report struct, implements `OutputFormatter` (`format_text` / `format_pretty`), and exposes a computation function. Rank-pattern commands (files, imports, ownership, docs, ceremony, surface, depth-map, layering, test-ratio, budget, density, coupling, uniqueness, module-health, call-complexity, hotspots, contributors) implement `RankEntry` on their entry structs and use `format_ranked_table()` from `normalize-rank::ranked` for shared tabular rendering. All rank commands also implement `DiffableRankEntry` for `--diff <ref>` support (compare current state against a git ref). The four scalar trend commands have been merged into `trend-metric --metric <complexity|length|density|test-ratio>`. `UnionFind` in `clusters.rs` is shared by `coupling_clusters.rs` and `fragments.rs`.

`liveness.rs` — `LivenessReport` and `analyze_liveness(idx, file, func)`: queries `cfg_blocks`, `cfg_edges`, `cfg_defs`, `cfg_uses` from the index, then runs a backward-dataflow fixed-point to produce `BlockLiveness` (live-in/live-out sets) per block. Powers `normalize analyze liveness <file> --function <name>`.

`exceptions.rs` — `ExceptionsReport` and `analyze_exceptions(idx, file, func)`: queries `cfg_edges` (kind=exception) and `cfg_blocks` from the index to produce per-function throw/catch summaries. Each `ThrowEntry` has a throw block, exception type (if captured), and the catch block IDs it routes to. Each `CatchEntry` shows the caught types and the number of throws handled. Unhandled throws (routing to the exit block) are flagged. Powers `normalize analyze exceptions <file> [--function <name>]`.

`graph.rs` — owns the graph report/presentation layer over the pure algorithms in `normalize-graph`. Defines `GraphTarget` (Modules/Symbols/Types), `GraphStats`, `GraphReport`, `DependentsReport`, their `OutputFormatter` impls, and `assemble_graph_report` (calls `find_sccs`/`find_diamonds`/`find_bridges`/`find_longest_chains`/`find_transitive_edges`/etc. and shapes the result). Also builds the call/type/import adjacency lists from the index. `normalize-graph` stays pure algorithms only; the normalize-flavored presentation lives here.

Representative files: `complexity.rs`, `length.rs`, `hotspots.rs`, `coupling.rs`, `coupling_clusters.rs` (queries `co_change_edges` from the structural index when available; falls back to git walk with a `tracing::warn!` when table is empty), `duplicates.rs`, `duplicates_views.rs`, `docs.rs`, `imports.rs`, `import_path.rs` (`ImportPathReport`, BFS over resolved import graph for `normalize view import-path <from> <to>`; supports `--all` for all simple paths and `--reverse` to find paths in the other direction), `layering.rs`, `ownership.rs`, `security.rs`, `trace.rs`, `trend.rs`, `trend_metric.rs`, `call_graph.rs` (`CallEntry` struct with optional `access` field for read/write distinction), `graph.rs`, `dependents.rs`, `fragments.rs`, `skeleton_diff.rs`, `summary.rs`. `git_utils.rs` is a thin re-export of `normalize_git` (`pub use normalize_git::*`). The implementation lives in the `normalize-git` crate; all public symbols are the same as before (open_repo, git_head, git_commit_timestamps, git_log_timestamps, resolve_ref, resolve_merge_base, git_show, git_diff_name_status, git_ls_files, git_remote_origin_url, git_activity_commits, git_file_churn_stats, git_per_commit_files, git_author_commit_counts, git_has_uncommitted_content_changes, git_summary_has_uncommitted_changes, git_last_commit_for_path, git_commit_count_for_path, format_unix_date, run_in_worktree, read_blob_text, read_blob_bytes, walk_tree_at_ref, FileChangeKind, FileChange, diff_base_to_head). `git_history.rs` provides shared worktree infrastructure (`run_in_worktree`, `resolve_ref`, `git_log_timestamps`, `select_snapshots`) used by `trend.rs` and `trend_metric.rs`. Native rule checks and the `gitignore_walk()` walker live in `normalize-native-rules`, not here. Progress bars (indicatif) are used in `duplicates.rs` and `architecture.rs` for long-running operations. Duplicate detection suppresses widespread body-pattern clusters and parallel-impl directory pairs to avoid O(N²) noise. `ast.rs` exports `build_ast_output(file, at_line, sexp, depth, compact)` — `depth` (-1 = unlimited) truncates the CST at that level, `compact` produces a node-type outline without source text via `tree_to_outline()`.
