# src/commands/analyze

All `normalize analyze` subcommand implementations (~44 files). Each file defines a report struct, implements `OutputFormatter` (`format_text` / `format_pretty`), and exposes a computation function. Representative subcommands: `complexity`, `complexity_trend`, `length`, `length_trend`, `hotspots`, `coupling`, `coupling_clusters`, `duplicates`, `docs`, `imports`, `layering`, `ownership`, `security`, `trace`, `trend`, `call_graph`, `graph`, `dependents`, `fragments`, `skeleton_diff`, `summary`. `mod.rs` declares all modules and defines the shared `AnalyzeConfig` struct. `git_history.rs` provides shared worktree infrastructure (`run_in_worktree`, `resolve_ref`, `git_log_timestamps`, `select_snapshots`) used by `trend.rs`, `complexity_trend`, and `length_trend`. Native rule checks (`stale_docs`, `stale_summary`, `check_refs`, `check_examples`) and the shared `gitignore_walk()` walker have been extracted into the `normalize-native-rules` crate. The `rules_cmd` module has been extracted to the `normalize-rules` crate. `dependents` supersedes the former `impact` command — it shows blast-radius data (depth, test coverage, fan-in) for the modules graph and a flat list for symbols/types.
