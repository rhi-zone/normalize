# src/serve

Server implementations that expose normalize functionality over network protocols. Modules: `http.rs` (HTTP REST API server, gated behind the `http` feature — pulls `axum` + `utoipa`), `lsp.rs` (Language Server Protocol server, gated behind the `lsp` feature — pulls `tower-lsp`; three-tier diagnostics: immediate per-file syntax rules on save, debounced workspace-wide fact rules with incremental index update, debounced workspace-wide native rules re-triggered on `.git/index` changes; persistent `SkeletonExtractor` avoids per-request recreation; configurable debounce via `[serve] fact_debounce_ms`), `mcp.rs` (Model Context Protocol server for LLM integration via stdio, real impl behind the `mcp` feature with an always-present `serve_mcp` entry that returns a runtime error when compiled out). `mod.rs` declares the transport modules (`http`/`lsp` behind their feature cfgs, `mcp` always with its internal stub) and defines `ServeConfig` (port/host/fact_debounce_ms from normalize.toml). The live dispatch is `service/serve.rs` (server-less `#[cli]`); each transport is a `default = true` capability surface reachable via the `serve` umbrella feature. All three protocols expose the same underlying code intelligence data.
