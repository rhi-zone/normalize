# src/serve

Server implementations that expose normalize functionality over network protocols. Modules: `http.rs` (HTTP REST API server), `lsp.rs` (Language Server Protocol server with three-tier diagnostics: immediate per-file syntax rules on save, debounced workspace-wide fact rules with incremental index update, debounced workspace-wide native rules re-triggered on `.git/index` changes; persistent `SkeletonExtractor` avoids per-request recreation; configurable debounce via `[serve] fact_debounce_ms`), `mcp.rs` (Model Context Protocol server for LLM integration via stdio). `mod.rs` defines `ServeConfig` (port/host/fact_debounce_ms from normalize.toml) and the `ServeArgs`/`ServeProtocol` clap types. The `mcp` feature flag gates MCP support. All three protocols expose the same underlying code intelligence data.
