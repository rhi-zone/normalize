# src (normalize crate source root)

Root of the normalize library and binary. `main.rs` handles argv[0] dispatch (symlink-as-drop-in for `rg`, `jq`, `sg`) and delegates to the server-less service layer or legacy clap dispatch. `lib.rs` declares all top-level modules.

**Key infrastructure:** `config.rs` (NormalizeConfig/TOML), `output.rs` (OutputFormatter trait + format dispatch), `diagnostic_convert.rs` (Finding/ABI diagnostic → Issue), `runtime.rs` (async helpers: `block_on`/`try_block_on` using `block_in_place` inside tokio, new runtime outside), `index.rs`, `symbols.rs`, `parsers.rs`, `extract.rs`, `skeleton.rs`, `tree.rs`.

**Naming convention:** All public output structs use the `*Report` suffix (not `*Result`). This applies uniformly: `LintListReport`, `LintRunReport`, `GrepReport`, `RebuildReport`, `PackagesReport`, `CommandReport`, `QueryReport`, `DaemonActionReport`, `DaemonRootReport`, `GenerateReport`, `GrammarInstallReport`, `InitReport`, `UpdateReport`, `TranslateReport`, `MeasureReport`, `AddReport`, `CheckReport`, `RemoveReport`.

**`service/`** is the primary CLI registration point (server-less `#[cli]` proc macro). Rule orchestration has been extracted into `normalize-rules`; `commands/rules.rs`, `service/rules.rs`, and `commands/analyze/rules_cmd.rs` have been removed. `service/facts.rs` — `FactsService` includes `query(sql)` for read-only SQL against the structural index; three SQL views (`entry_points`, `external_deps`, `external_surface`) are created at index open time. `service/config.rs` — `ConfigService` provides `normalize config schema/show/validate/set` with TOML/JSON/YAML inspection and `jsonschema` validation. `service/ratchet.rs` and `service/budget.rs` mount `normalize-ratchet::RatchetService` and `normalize-budget::BudgetService`. `service/guide.rs` — `GuideService` with 5 built-in workflow guides. `daemon.rs` — `Event` enum (`FileChanged`, `IndexRefreshed`), `Request::Subscribe`, `DaemonClient::watch_events()` for streaming event subscriptions; `WatchedRoot` holds an in-memory reverse-dep graph updated incrementally on each refresh.

**`commands/`** contains domain logic modules. Key areas: `commands/analyze/` (AST/complexity/ownership/coupling analysis, tree-sitter introspection via `ts_parse.rs`/`ts_query.rs`/`ts_node_types.rs`), `commands/sessions/` (session analysis, Markov transition matrices, subagent discovery), `commands/tools/` (linter/formatter/type-checker integration).
