# src (normalize crate source root)

Root of the normalize library and binary. `main.rs` handles argv[0] dispatch (symlink-as-drop-in for `rg`, `jq`, `sg`) and delegates to the server-less service layer or legacy clap dispatch. `lib.rs` declares all top-level modules.

**Key infrastructure:** `config.rs` (NormalizeConfig/TOML), `output.rs` (OutputFormatter trait + format dispatch), `diagnostic_convert.rs` (Finding/ABI diagnostic → Issue), `runtime.rs` (async helpers: `block_on`/`try_block_on` using `block_in_place` inside tokio, new runtime outside), `index.rs`, `symbols.rs`, `parsers.rs`, `extract.rs`, `skeleton.rs`, `tree.rs`.

**Naming convention:** All public output structs use the `*Report` suffix (not `*Result`).

**`service/`** is the primary CLI registration point (server-less `#[cli]` proc macro). Key services: `view.rs` (file/dir/symbol navigation; `ViewReport.summary` populated from SUMMARY.md for dirs and from module-level doc comments for files), `rename.rs` (`normalize rename` — cross-file symbol rename with `--dry-run`, structured `RenameReport`), `rank.rs` (20+ ranking commands), `syntax.rs` (`syntax ast` with `--depth`/compact outline; `syntax query` compact shows per-match locations), `rules.rs` (mounts `normalize-rules::RulesService`; `compile` validates `.dl` files; `show`/`tags` return structured JSON), `facts.rs` (SQL query against structural index), `config.rs` (`normalize config schema/show/validate/set`), `ratchet.rs`/`budget.rs`, `guide.rs`, `daemon.rs` (Unix domain sockets, ENGINE_CACHE warming, reverse-dep graph; stub on Windows), `package.rs` (`PackageListReport.ecosystems_detected`).

**`commands/`** contains domain logic modules. Key areas: `commands/analyze/` (AST/complexity/ownership/coupling/architecture analysis; `analyze health --limit` caps `large_files`; `analyze architecture --limit` caps `cross_imports`), `commands/view/` (unified `ViewReport` with `summary` for both dirs and files; `render_line_range` header uses `strip_range_suffix` to avoid duplicating `:N-M` in file:range requests), `commands/sessions/` (session analysis, Markov matrices; `sort.rs` provides shared `SortDir`/`SortKey<F>`/`SortSpec<F>`/`DefaultDir` used by `list`, `messages`, `stats`, and `patterns` for `--sort` support), `commands/tools/` (linter/formatter/type-checker integration). `text_search.rs` (`GrepReport` output groups consecutive same-symbol matches under a single header to avoid repeating symbol tags per line).
