# normalize (main binary crate)

The primary CLI binary crate that wires together all normalize sub-crates into the `normalize` command. It depends on ~30 domain crates (`normalize-facts`, `normalize-languages`, `normalize-edit`, `normalize-session-analysis`, etc.) and exposes them through a unified service layer. Three embedded CLI tools are included as optional features: `rg` (ripgrep), `jq` (jaq), and `ast-grep`/`sg`, each invocable via subcommand or via symlink to the binary.

The `src/commands/` directory holds domain logic (analysis, view, etc.); all `cmd_*` i32-returning wrappers eliminated from both `commands/` and `normalize-rules/src/` — service methods return `Result<T, String>` with `?` propagation. The `src/service/` directory is the server-less `#[cli]`-annotated registration layer. `config show --set-only` hides unset fields; `config set` now blocks on schema errors (use `--force` to bypass). Multi-repo support standardized across all analyze and tools commands. `complexity --diff <ref>` / `length --diff <ref>` show per-function deltas; `complexity-trend` / `length-trend` show metric over history. `normalize facts` renamed to `normalize structure` (field rename in service/mod.rs). LSP diagnostics now two-tier: immediate per-file syntax, debounced workspace-wide fact rules. Daemon calls `incremental_call_graph_refresh()` after detecting changes. `normalize sessions grep` added: line-granularity search across session messages with context lines. `rules recommended` field added; correctness rules enabled by default; standalone `rules setup` command. `normalize view` refactored (phase 2): `ViewOutput` enum and all per-kind report structs dissolved into a single `ViewReport` struct (unified `ViewNode` tree); `normalize view list` added as a new subcommand returning `ViewListReport(Vec<ViewReport>)` for multi-result views. Updated 2026-03-16.
