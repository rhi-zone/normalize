# normalize (main binary crate)

The primary CLI binary crate that wires together all normalize sub-crates into the `normalize` command. It depends on roughly 30 domain crates (`normalize-facts`, `normalize-languages`, `normalize-edit`, `normalize-session-analysis`, etc.) and exposes them through a unified service layer built with the server-less `#[cli]` proc macro. Three embedded CLI tools are included as optional features: `rg` (ripgrep), `jq` (jaq), and `ast-grep`/`sg`, each invocable via subcommand or via symlink to the binary. The default feature is `cli-full = ["cli", "jq-cli", "rg-cli", "ast-grep-cli"]`.

The `src/commands/` directory holds domain logic (report structs, `OutputFormatter` impls, computation functions) organized by feature area. The `src/service/` directory is the server-less `#[cli]`-annotated registration layer; service methods return `Result<T, String>` with `?` propagation. The root `NormalizeService` composes sub-services: `AnalyzeService`, `ConfigService`, `DaemonService`, `EditService`, `FactsService`, `GrammarService`, `GuideService`, `GenerateService`, `PackageService`, `RankService`, `RatchetService`, `RulesService` (from `normalize-rules`), `ServeService`, `SessionsService`, `SyntaxService`, `ToolsService`, and `ViewService`. `main()` is `#[tokio::main] async fn`; service methods are `async fn`.
