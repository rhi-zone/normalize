# normalize (main binary crate)

The primary CLI binary crate that wires together all normalize sub-crates into the `normalize` command. It depends on ~30 domain crates (`normalize-facts`, `normalize-languages`, `normalize-edit`, `normalize-session-analysis`, etc.) and exposes them through a unified service layer. Three embedded CLI tools are included as optional features: `rg` (ripgrep), `jq` (jaq), and `ast-grep`/`sg`, each invocable via subcommand or via symlink to the binary.

The `src/commands/` directory holds domain logic (analysis, view, etc.); all `cmd_*` i32-returning wrappers eliminated from both `commands/` and `normalize-rules/src/` — service methods return `Result<T, String>` with `?` propagation. The `src/service/` directory is the server-less `#[cli]`-annotated registration layer. Multi-repo support (`--repos-dir`) standardized across all analyze and tools commands (ownership, hotspots, duplicates, contributors, activity, repo_coupling, cross_repo_health, tools lint/test). `type_refs` extraction extended to C#, Kotlin, Swift, C++, Ruby. Native rule checks extracted into `normalize-native-rules`; rule orchestration into `normalize-rules`. Updated 2026-03-09.
