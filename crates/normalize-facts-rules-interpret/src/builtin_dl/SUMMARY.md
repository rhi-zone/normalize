# normalize-facts-rules-interpret/src/builtin_dl

Built-in Datalog rule files (`.dl`) for architectural analysis, bundled with the interpreter crate.

Each file contains a TOML frontmatter header (rule id, message, enabled flag) followed by Ascent Datalog rules. Rules: `barrel_file.dl`, `bidirectional_deps.dl`, `circular_deps.dl`, `dead_api.dl`, `deep_nesting.dl`, `duplicate_symbol.dl`, `fan_out.dl`, `god_class.dl`, `god_file.dl`, `hub_file.dl`, `layering_violation.dl`, `liveness.dl`, `missing_export.dl`, `missing_impl.dl`, `missing_test.dl`, `orphan_file.dl`, `resolution.dl`, `self_import.dl`, `stale_mock.dl`, `unused_import.dl`. All rules use `diagnostic(severity, rule_id, file, line, message)`; file-level rules set both `file` and `message` to the path, symbol-level rules set `file`+`line` from the `symbol` relation and put the symbol name in `message`; `hub-file` has no file location (uses module name in `message`). `unused_import.dl` suppresses Rust/TS/TSX/SCM/fixture files (indexer false positives: type/JSX/namespace usage not captured by `call` relation). `missing_test.dl` and `stale_mock.dl` are disabled by default and use attribute-based detection (interpreter does not support string method calls like `.starts_with()`/`.contains()`). `resolution.dl` is disabled by default and derives `resolved_reference` and `resolved_call` from Phase 0 cross-file resolution predicates. `liveness.dl` is disabled by default and computes standard backward-dataflow live-in/live-out variable sets per CFG block using `cfg_block`, `cfg_edge`, `cfg_def`, `cfg_use` relations. `effects.dl` is disabled by default and derives `async_function`, `defer_function`, `generator_function`, `resource_acquire`, `resource_leak` from the `cfg_effect` relation (Phase 3). `exception_flow.dl` is disabled by default and derives `exception_reaches`, `unhandled_exception`, `can_throw` from typed `cfg_edge` exception edges (Phase 4).
