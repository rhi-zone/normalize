# normalize-native-rules

Native rule check implementations for normalize. Provides built-in checks that run without tree-sitter parsing: `stale-summary` / `missing-summary` (SUMMARY.md freshness via git history), `broken-ref` (broken symbol references in markdown docs), `stale-doc` (documentation with stale `covers:` annotations), `missing-example` (missing example references in docs), and `ratchet/complexity-total` / `ratchet/complexity-per-function` (metric regression checks against `.normalize/ratchet.json`). All checks produce `DiagnosticsReport` (from `normalize-output`) and are called by the native engine in `normalize rules run`. `NATIVE_RULES` exports static `NativeRuleDescriptor` metadata for `normalize rules list`.
