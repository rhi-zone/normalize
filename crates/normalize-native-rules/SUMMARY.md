# normalize-native-rules

Native rule check implementations for normalize. Provides six built-in checks that run without tree-sitter parsing: `missing-summary` (directory has no committed doc file), `stale-summary` (doc file exists but is stale relative to directory commits), `broken-ref` (broken symbol references in markdown docs), `stale-doc` (documentation with stale `covers:` annotations), and `missing-example` (missing example references in docs). Both summary rules accept a `filenames` config (alternate doc filenames, OR semantics) and a `paths` config (directory glob filter — empty means check everywhere). All checks produce `DiagnosticsReport` (from `normalize-output`) and are called by the native engine in `normalize rules run` and `normalize analyze check`. `NATIVE_RULES` exports static `NativeRuleDescriptor` metadata for `normalize rules list`. Regex patterns in `check_examples.rs` and `check_refs.rs` are compiled once via `std::sync::OnceLock` statics. Uses `gix` (pure-Rust) for HEAD commit hash resolution in `stale_summary.rs`; other git operations remain as shell-outs where gix has no equivalent.
