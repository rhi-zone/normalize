# normalize-native-rules

Native rule check implementations for normalize. Provides five built-in checks that run without tree-sitter parsing: `stale-summary` / `missing-summary` (SUMMARY.md freshness via git history), `broken-ref` (broken symbol references in markdown docs), `stale-doc` (documentation with stale `covers:` annotations), and `missing-example` (missing example references in docs). All checks produce `DiagnosticsReport` (from `normalize-output`) and are called by the native engine in `normalize rules run` and `normalize analyze check`. `NATIVE_RULES` exports static `NativeRuleDescriptor` metadata for `normalize rules list`.
