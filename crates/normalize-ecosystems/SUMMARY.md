# normalize-ecosystems

Project dependency management for multiple package ecosystems.

Defines the `Ecosystem` trait for detecting, querying, and auditing project dependencies. Provides types `PackageInfo`, `Dependency`, `DepSource`, `DependencyTree`, `Vulnerability`, and `AuditResult` — all now deriving `schemars::JsonSchema` so they can be embedded in typed CLI report structs. Built-in implementations cover 12 ecosystems (cargo, npm, deno, python, go, hex, gem, composer, maven, nuget, nix, conan) selected via Cargo features, plus a global plugin registry (`detect_ecosystem`, `get_ecosystem`, `register_ecosystem`). The `query()` method on `Ecosystem` uses a 24-hour on-disk cache and falls back to stale cache on network failure. All public fields carry `///` doc comments describing their semantics.

For the `normalize docs` command the crate also carries symbol-documentation infrastructure: `SymbolDoc`/`DocFormat` types; Rust extractors (`local_docs`, `docs_rs`); Go extractors (`go_docs`, gated behind the `go` feature — module-cache/SDK local extraction + module-proxy `.zip` remote fetch); Python extractors (`python_docs`, gated behind the `python` feature — `site-packages`/stdlib local extraction + PyPI sdist remote fetch); shared multi-language primitives `source_archive` (download + extract package source archives to the cache) and `doc_tree` (tree-sitter-based docstring/signature extraction from an extracted source tree); and `http` (the ureq wrapper used to download registry archives). The `Ecosystem` trait exposes docs generically via `docs_extractor` / `docs_fetcher` / `package_from_symbol` / `docs_language`. These depend on `normalize-languages` (the `Language` trait + parsers) and `normalize-local-deps`.
