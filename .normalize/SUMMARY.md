# .normalize

Project-local normalize configuration and state. `config.toml` holds rule configuration, thresholds, feature flags, and per-command allow lists (formerly stored as separate `*-allow` files, now consolidated under `[rules.rule."<id>"] allow` and `[analyze.<subcommand>] allow` sections). `facts.db`/`index.db`/`index.sqlite`/`memory.db` are index databases. `syntax-cache.json` caches syntax rule results per file. `cache/` holds incremental analysis caches. `rules/` and `scripts/` contain custom rule definitions and helper scripts. `shadow/` stores shadow analysis data. `agent/` holds agent-related state. `benchmark-results.txt` records performance measurements.
